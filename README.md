## 1. 摘要 (Executive Summary)

KILO Network 旨在打破传统 VPN 服务的中心化格局。通过利用 **Solana** 的高性能账本、**libp2p** 的对等网络技术以及 **10 级级联分润机制**，KILO 构建了一个用户间直接相互代理的物理网络。在 KILO 生态中，每个用户设备既是流量的消费者，也是加密代理的提供者，通过 100 亿枚 KILO 代币实现全球带宽资源的公平定价与高效流转。

## 3. 技术实现架构 (Technical Implementation)

### 3.1 用户间相互代理模式 (U2U Proxying)
KILO 彻底取消了中心化中继，采用完全对称的节点设计：
* **Provider Mode**：用户共享闲置带宽，设备变身为高性能出口。
* **Consumer Mode**：用户支付 KILO 币，从全球住宅 IP 池中随机或定向匹配代理。

### 3.2 混合隧道驱动 (Multi-Tunnel Fusion)
内核集成多种协议，支持根据网络环境自动切换：
* **WireGuard (UDP)**：采用 ChaCha20 加密，提供接近线速的吞吐，适合游戏与 4K 流媒体。
* **Trojan/TLS (TCP)**：将加密流量封装在合法 TLS 握手中，对抗深度包检测 (DPI)。
* **NAT 穿透**：集成 STUN/ICE 技术，解决 90% 家庭宽带无公网 IP 的对等连接难题。



### 3.3 活跃证明 (Proof of Liveness)
心跳包不仅是活跃信号，更是收益结算的凭证：
1.  **发送**：客户端每 60 秒向邀请链上的 10 个上线发送 `SignedHeartbeat`。
2.  **验证**：上线节点在本地缓存下线的活跃权重。
3.  **结算**：若某时段心跳缺失，该层级的分润将自动触发销毁机制。

## 4. 代币经济学 (Tokenomics)

### 4.1 代币分发
* **总量**：10,000,000,000 KILO (100 亿枚，恒定)。
* **用途**：流量计费、节点质押、邀请分润、治理投票。

### 4.2 收益分配矩阵 (70/30 规则)
当消费者支付 $X$ 金额的 KILO 币时，系统自动执行实时分润：

* **70% -> 卖家收益**：直接结算给实际提供代理服务的用户。
* **30% -> 邀请池收益**：
    | 层级 | 比例 (总额 %) | 角色 |
    | :--- | :--- | :--- |
    | L1 (直接邀请人) | 15.0% | 核心拉新奖励 |
    | L2 | 5.0% | 二级管理收益 |
    | L3 - L5 | 各 2.0% | 社区密度维护 |
    | L6 - L10 | 各 0.8% | 长尾裂变奖励 |

### 4.3 动态计费权重 ($W_p$)
$$Total\_Cost = (GB \times Base\_Rate) \times W_p$$
* **WireGuard**: $W_p = 1.0$
* **Trojan/V2Ray**: $W_p = 1.5$ (补偿混淆产生的额外 CPU/算力成本)

## 5. 安全与隐私防护

### 5.1 地址安全性
用户使用自定义字符串作为种子（Seed），通过 **BIP-39** 标准派生 Solana 钱包。由于私钥由字符串生成，用户需自行妥善保管字符串。

### 5.2 流量安全隔离
* **内核级沙箱**：转发流量运行在独立的网络命名空间，严禁访问本地文件系统及内网设备。
* **黑名单审计**：代理提供者可一键开启“合规过滤”，阻断高风险协议（如 SMTP 垃圾邮件），保护自身 IP 信用。
